Introduction

This notebook develops a binary classification model to predict loan default risk for Home Credit Group applicants. The target variable TARGET equals 1 for applicants who experienced payment difficulties and 0 otherwise. The dataset is heavily imbalanced (~8% default rate), so we use AUC-ROC as our primary evaluation metric rather than accuracy.

The modeling workflow:

  1. Baseline benchmark — majority class classifier and a naive logistic regression
  2. Candidate model comparison — logistic regression variants, random forest, LightGBM
  3. Class imbalance handling — SMOTE, upsampling, downsampling, and class weights
  4. Hyperparameter tuning — randomized search on a 5K subsample with 3-fold CV
  5. Supplementary features — bureau, previous application, and installment aggregations
  6. Final model — retrain best configuration on full training data

Key Findings

  1. Baselines confirmed class imbalance challenge. The majority class classifier scored ~0.50 AUC despite ~92% accuracy, confirming that accuracy is a misleading metric. The EXT_SOURCE-only logistic regression immediately provided meaningful lift.
  2. LightGBM dominated the candidate comparison. Gradient boosting consistently outperformed logistic regression and random forest on AUC.
  3. Class imbalance handling provided modest but consistent gains. scale_pos_weight was the simplest and most effective strategy.
  4. Hyperparameter tuning improved AUC over defaults. Randomized search on a 5K subsample efficiently found better settings.
  5. Supplementary features provided additional lift. Bureau credit history, previous application behavior, installment payment patterns, POS cash balances, and credit card utilization added meaningful signal.

Business Implications

The final model produces predicted probabilities that can be used to rank applicants by default risk. The model substantially outperforms the majority class benchmark, meaning Home Credit can more confidently extend credit to lower-risk applicants with limited credit histories — directly addressing the stated business problem.
